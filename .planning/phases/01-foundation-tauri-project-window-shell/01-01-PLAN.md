**Step 1 — Install Rust (BUILD-01):**

Run: `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y`
Then: `source "$HOME/.cargo/env"` and verify `rustc --version` >= 1.77.2.
Also verify Xcode CLI tools: `xcode-select -p` (should return a path). If not installed, run `xcode-select --install`.

**Step 2 — Scaffold Tauri v2 project (BUILD-01):**

From the project root `/Users/tomostapchuk/Desktop/Claude/Top Focus`, run:
```
bun create tauri-app . --template vanilla --manager bun
```

If the scaffold prompts interactively or fails because the directory is non-empty, manually create the structure:
- Create `package.json` with name `top-focus`, scripts `{"tauri": "tauri"}`
- Run `bun add @tauri-apps/cli @tauri-apps/api`
- Run `bun add @tauri-apps/plugin-store @tauri-apps/plugin-autostart @tauri-apps/plugin-global-shortcut @tauri-apps/plugin-positioner`
- If `src-tauri/` was not created by scaffold, run `bunx tauri init` (answer: app name = "Top Focus", window title = "Top Focus", frontend dev URL = `http://localhost:1420`, frontend dist = `../src`)

**Step 3 — Configure Rust dependencies in Cargo.toml (BUILD-02):**

Ensure `src-tauri/Cargo.toml` contains these dependencies under `[dependencies]`:
```toml
tauri = { version = "2", features = ["macos-private-api"] }
tauri-plugin-store = "2"
tauri-plugin-autostart = { version = "2", features = ["macos"] }
tauri-plugin-global-shortcut = "2"
tauri-plugin-positioner = { version = "2", features = ["system-tray"] }
objc = "0.2"
```

The `objc` crate is needed for Plan 01-02 (NSPanel non-activating behavior). Install it now to avoid recompilation later. The `macos-private-api` feature on tauri is REQUIRED for transparent windows.

**Step 4 — Register all plugins in lib.rs (BUILD-02):**

Edit `src-tauri/src/lib.rs` to register plugins:
```rust
use tauri_plugin_autostart::MacosLauncher;

pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_store::Builder::new().build())
        .plugin(tauri_plugin_autostart::init(
            MacosLauncher::LaunchAgent,
            Some(vec![]),
        ))
        .plugin(tauri_plugin_global_shortcut::Builder::new().build())
        .plugin(tauri_plugin_positioner::init())
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

Ensure `src-tauri/src/main.rs` calls `top_focus_lib::run()` (or whatever the crate name is based on `Cargo.toml` `[package].name` — likely `top-focus` which becomes `top_focus`). Check the generated main.rs — Tauri v2 scaffold typically creates this correctly.

```json
      "icons/icon.icns",
      "icons/icon.ico"
    ],
    "macOS": {
      "minimumSystemVersion": "10.15",
      "signingIdentity": null,
      "providerShortName": null,
      "entitlements": null,
      "infoPlist": {
        "LSUIElement": true
      }
    }
  }
}
```

Key points:
- Window size is 348x108 (300+48 width, 60+48 height) to accommodate 24px CSS shadow on each side (per WIN-05, UI-SPEC).
- `macOSPrivateApi: true` — required for `transparent: true` to work.
- `LSUIElement: true` in infoPlist — hides from Dock and Cmd+Tab (WIN-04).
- `alwaysOnTop: true` — floating window (WIN-01).
- `decorations: false` + `transparent: true` — frameless transparent window (WIN-05).
- `withGlobalTauri: true` — exposes `window.__TAURI__` for vanilla JS without bundler.
- `shadow: false` — we use CSS shadow instead.
- `signingIdentity: null` — placeholder for BUILD-04; signing will be configured when Apple Developer certificate is available.

Note: If the scaffold creates a `devUrl` setup expecting a dev server and `src/` is served statically, set `"beforeDevCommand": ""` and `"devUrl": ""` (or remove devUrl). For vanilla HTML served from `../src`, Tauri can serve it directly. Check if this works — if `bun run tauri dev` complains about devUrl, adjust to serve files statically by removing devUrl entirely and only using `frontendDist`.

**Create `src/index.html`:**

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
  <title>Top Focus</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <div class="window-canvas">
    <div class="widget" data-tauri-drag-region>
      <span class="task-text" data-tauri-drag-region>What's your #1?</span>
    </div>
  </div>
  <script src="main.js" type="module"></script>
</body>
</html>
```

Key points:
- `viewport` meta disables zoom (PERF-04 prep, prevents layout breaks).
- `data-tauri-drag-region` on both `.widget` and `.task-text` — the entire compact surface is draggable (WIN-06). The span needs the attribute too so it does not block drag events from the parent.
- No framework, no bundler — vanilla HTML served directly by Tauri.
